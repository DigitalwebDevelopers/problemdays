Day 57 JavaScript Challenge — “Detect Task Cycles”

Taught By Code with Deepanshu Knox

🔍 Problem Of the Day —

You are building a project management system where tasks can depend on other tasks. A cycle occurs when a task indirectly depends on itself, causing an infinite loop. Detect if such a cycle exists in the task dependency graph.

🧪 Example Input: 

const dependencies = {
  taskA: ["taskB"],
  taskB: ["taskC"],
  taskC: ["taskA"]
};

🔸 Your Task to Perform

  • Write a function to detect if there is a **circular dependency** in the input.
  • Use **DFS (Depth-First Search)** and track visited & recursion stack.
  • Return true if a cycle exists; otherwise, return false.

🧪 Example Output: 

true // Because taskA → taskB → taskC → taskA

🔸 Bonus Tip for You

  • Consider writing a recursive helper function with `visited` and `recStack` sets.
  • This logic mimics **topological sorting** — think of it as validating a task pipeline.
