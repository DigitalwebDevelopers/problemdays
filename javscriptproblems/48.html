Day 48 JavaScript Challenge — “Secure PIN Pad Validation”

Taught By Code with Deepanshu Knox

🔍 Problem Of the Day —

You're building the logic behind a digital PIN pad for an ATM machine. For extra security, the system **randomly scrambles digits** on the screen every time, and stores the original digit-to-button mapping.

You receive a **scrambled mapping** and a **user click sequence**. Validate whether the sequence actually matches the correct PIN.

🧪 Example Input:

const correctPIN = "3481";

const pinMapping = {
  A: "3",
  B: "4",
  C: "8",
  D: "1",
  E: "5",
};

const userPressed = ["A", "B", "C", "D"];

🔸 Your Task to Perform

  • Convert user’s click sequence into actual number string using the mapping.
  • Compare this generated number string with the original PIN.
  • Return "Access Granted" if it matches, otherwise "Access Denied".

🧪 Example Output:

Access Granted

🔸 Bonus Tip for You

Ensure the mapping is one-way only. Don't reverse it.

Think about edge cases like missing keys, wrong lengths, or extra clicks.
